# 0.36.4 patch notes

## Fixes

- **Compact mode: collapse now ignores first hover activation** (`39ebeab`)
  Clicking collapse immediately re-showed the sidebar if the mouse was over it.
  Root cause: `pointer-events:none` applies before the 250ms transform transition,
  firing a synthetic `mouseleave` that cleared `_ignoreNextHover` too early.
  Now cleared via `transitionend` on `transform` + 400ms safety timeout.
  `setHover()` also actively suppresses show attempts while the flag is set.

- **Session restore (Ctrl+Shift+T): correct tab order and hierarchy** (`pending`)
  Multiple bugs caused tree hierarchy corruption on manual session restore:
  1. FIFO queue matching consumed stale entries after restore completed, so new
     tabs or lazy-tab activations overwrote correct assignments with wrong data.
  2. `onTabRestoring` correction path overwrote FIFO-assigned data with stale
     duplicate entries from the save file, changing tab IDs and orphaning parentId
     references (tab's parentId pointed to its own old ID → level 0 → flat tree).
  3. `sessionstore-windows-restored` observer was one-shot — caught startup restore
     but missed subsequent Ctrl+Shift+T restores, so the queue was never cleared.
  4. Save file accumulated duplicate leftover entries across sessions.
  Fix:
  - Gate FIFO matching with `_inSessionRestore` flag (true during startup and
    manual restore, false after `sessionstore-windows-restored`)
  - Clear `savedTabQueue` at `sessionstore-windows-restored` to prevent stale
    corrections on lazy-tab activation
  - Make observer persistent (fires for both startup and manual restores)
  - Add `sessionstore-initiating-manual-restore` observer to re-arm queue from
    `_lastLoadedNodes` for Ctrl+Shift+T, filtering out still-alive tabs by URL
  - Guard `onTabRestoring` correction path with `appliedSavedState` flag — FIFO
    assignments are authoritative and cannot be overwritten by stale queue entries
  - Filter duplicate URLs from leftover entries at save time

- **`gBrowser.moveTabTo` API fix / drag-drop regressions** (`fc8fa20`)
  `gBrowser.moveTab` does not exist in current Firefox — caused "moveTab is not a
  function" console errors and broken tab ordering during drag-drop reordering and
  pinned-tab placement. Fixed both call sites in `executeDrop` and the pinned-tab path.

- **Debug log flush race** (`pending`)
  `pfxLog` set `_logFlushPending = false` before the async I/O resolved, allowing
  concurrent flushes to race and duplicate or lose log lines. Now holds the flag
  until write completes and re-drains if new lines accumulated during I/O.
